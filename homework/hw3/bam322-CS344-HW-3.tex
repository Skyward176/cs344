\begin{problem}
  You are given an array of characters $S[1:n]$ such that each $S[i]$ is a small case letter from the English alphabet.
  You are also provided with a black-box algorithm $dict$ that given two indices $i,j \in [n]$, $dict(i,j)$ returns whether the sub-array $S[i:j]$ in array $S$ forms a valid word in English or not in $O(1)$ time. (Note that this algorithm is provided to you and you
  do not need to implement it in any way).

  Design and analyze a dynamic programming algorithm to find whether the given array $S$ can be partitioned into a sequence of valid words in English. The  runtime of your algorithm should be $O(n^2)$.

  \textbf{Example:} Input Array: $S=[maytheforcebewithyou]$.

  Assuming the algorithm $dict$ returns that $may, the, force, be, with$ and  $you$ are valid words (this means that for instance, for $may$ we have $dict(1,3)=\text{True}$), this array can be partitioned into a sequence of valid words.

\end{problem}

\begin{solution}
  \

  \textbf{Algorithm:}
  \
  %=================================================%
  %               1. THE ALGORITHM                  %
  %=================================================%
  \

  We set up a memo array of length n+1. The last value is true and represents an empty string that is splittable into zero valid words.

  Now we do this:

  \begin{verbatim}
    for i = n to zero:
      for j up to i:
        if dict(i,j) == true and (table[j+1] ==true)
          table[i] = true 
  \end{verbatim}

  These loops check if there is a valid split of the last n-i characters. 

  The result will be in table[0]. 
  %================================================%
  %            2. PROOF OF CORRECTNESS             %
  %================================================%
  \

  \textbf{Proof:}
  \

  \begin{itemize}
    \item \textbf{Base Case: } For an empty string, it must be splittable into zero valid words. 
    \item \textbf{Inductive Step: } For a string from i to n, it is splittable if there is a value j where the string from i to j is a valid word and the string after j to n is splittable.

    In the most trivial case. A string of length one (like the letter 'a') is splittable because the substring from 0 to 0 is a word 'a' and the substring after ("") is splittable.

    As we step that boundary down further and further, we will keep finding substrings of the larger array that can be perfectly partitioned into valid words. If this propagates down to the first character, then we know the whole string is splittable. The table in this case saves us the work of having to recalculate if every substring to the right of the boundary is splittable, instead giving a memo of what substrings were splittable.
  \end{itemize}
  \

  \textbf{Runtime:}
  \
  %================================================%
  %            3. RUN TIME ANALYSIS                %
  %================================================%
  The main work of this occurs in two loops over the whole of the string, giving $O(n^2)$ complexity
\end{solution}

\begin{problem}

  Suppose you are given an array $A[1:n]$ of integers that can be positive or negative. A \emph{contiguous} subarray $A[i:j]$ of $A$ is called a \textbf{positive} subarray if sum of its entries is strictly positive, i.e.,
  \[
    \sum_{k=i}^{j} A[k] > 0.
  \]
  Design and analyze a dynamic programming algorithm that in $O(n^2)$ time finds the smallest number of positive contiguous subarrays of $A$ such that every positive entry of $A$ belongs to exactly one of these subarrays. You can assume that $A$ has at least one positive entry and thus the answer is \emph{at least} one.

  \medskip

  \paragraph{Example:} The correct answer for the following array is $3$ subarrays.

  \begin{figure}[h!]
    \centering
    \includegraphics[width=0.65\textwidth]{positive-cover.png} \hspace{-0.15cm}
  \end{figure}

\end{problem}

\begin{solution}

  %=================================================% 1. THE ALGORITHM %=====================================================
  \
  
  \begin{verbatim}
    def function(A[n])
      memo = [0:n]
      memo[0] = 0
      memo[1:n] = +infty
      for i in range 1:n:
        if A[i] <= 0:
          memo[i] = memo[i-1]
      
        cur = 0 
        for j in (i down to 1):
          cur = cur + A[j]
          if cur > 0 and memo[i-1] != +infty:
            memo[i] = min(memo[i], memo[i-1] + 1)
      return memo[n] 
  \end{verbatim}

  %=================================================
  % 2. PROOF OF CORRECTNESS %=====================================================
  \begin{enumerate}
    \item \textbf{Base: } An empty array requires 0 positive contiguous subarrays. 
    \item \textbf{Inductive Step: } 
    \

    If we assume that the memo array stores correct solutions for the array from 0 to an index i, then this is what happens when we calculate memo[i+1]:
    \

    If A[i+1] is negative, it doesn't affect our solution and we can carry over the answer from a[i]. So memo[i+1] = memo[i]

    By taking the running sum from the index down to 0, we determine that the subsequent indices are part of the same subarray and therefore have the same memoized solution. Only when the current index we're adding makes the solution smaller does it change. 

    So either the new value forms part of one of the existing subarrays and the previous value gets carried forward, or it doesn't and we add one to the solution at that index.
    \
    \item  
  \end{enumerate}
  %=================================================
  % 3. RUN TIME ANALYSIS %=====================================================

  This has two nested loops that iterate over the array, so its $O(N^2)$

\end{solution}

\begin{problem}
  We define a number to be \textbf{special} if it can be written as:
  \[
    n = a \cdot 197 + b \cdot 232
  \]
  where \(a\) and \(b\) are non-negative integers. For example:
  \begin{itemize}
    \item \(696\) is special because it can be written as \(696 = 0 \cdot 197 + 3 \cdot 232\).
    \item \(2412\) is special because it can be written as \(2412 = 4 \cdot 197 + 7 \cdot 232\).
    \item \(267\) is not special (since it cannot be written as \(a \cdot 197 + b \cdot 232\) for non-negative \(a\) and \(b\)).
  \end{itemize}

  The goal of this problem is to write a dynamic programming algorithm \textbf{Special(n)} that, given a positive integer \(n\), either finds non-negative integers \(a\) and \(b\) such that \(n = a \cdot 197 + b \cdot 232\), or returns “no solution” if no such \(a\) and \(b\) exist.

  Remember to separately write your specification of recursive formula in plain English , your recursive solution for the formula and its proof of correctness , the algorithm using either memoization or bottom-up dynamic programming , and runtime analysis.

\end{problem}

\begin{solution}

  %=================================================
  % 1. THE ALGORITHM %=====================================================
  \
    
  \begin{verbatim}
    def special(n):
      array products;
      products[0] = (0,0)

      for i in range(1, n + 1):
        if i >= 197 and products[i - 197] is not None:
            a, b = products[i - 197]
             products[i] = (a + 1, b)
              
          elif i >= 232 and products[i - 232] is not None:
              a, b = products[i - 232]
              products[i] = (a, b + 1)
              
      # Check if we found a valid combination for the target n
      if products[n] is not None:
          return products[n]
      else:
          return "no solution"
  \end{verbatim}
  %=================================================
  % 2. PROOF OF CORRECTNESS %=====================================================
  \begin{enumerate}
    \item \textbf{Base: } The number zero can be formed with a =0 and b= 0.
    \item \textbf{Inductive Step: } If any number is special, then subtracting 197 or 232 will give a smaller special number. Conversely, counting up by 197 and 232 will give us every subsequent special number. 0+197 = 197, 197 + 197 = 2(197), 197 + 232, etc.
  \end{enumerate} 
  %=================================================
  % 3. RUN TIME ANALYSIS %=====================================================

  This is going to run in logn times, since the max number of times it would take to confirm that a number isn't special is $log_{197}(n) * log_{232}(n)$

\end{solution}
